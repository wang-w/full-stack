## 背景

一位拥有10年经验的前端开发工程师需要转型到Java后端开发。公司使用Java作为后端服务技术栈。本设计建立了一个渐进式学习路径，从Java基础知识开始，直至能够开发生产级的Spring Boot应用程序。

当前状态：拥有扎实的前端基础，无Java后端经验。
约束条件：必须符合企业级Java实践（Spring Boot、Maven/Gradle、JPA）。
相关方：准备转型后端开发角色的个人学习者。

## 目标 / 非目标

**目标:**
- 提供从Java基础到Spring Boot精通的结构化学习路径
- 构建展示核心后端模式的完整RESTful API项目
- 涵盖必要的企业级Java概念（依赖注入、ORM、安全）
- 为每个学习模块创建可运行的代码示例
- 建立配备适当工具的开发环境（IDE、构建工具、调试）

**非目标:**
- 高级分布式系统（微服务、服务网格）- 先专注于单体应用
- 云原生部署模式（Kubernetes、容器）- 仅在部署基础中涵盖
- 深入Java内部机制（JVM调优、字节码）- 初期能力培养不需要
- 前端集成 - 纯粹专注于后端API开发
- 多种数据库类型 - 专注于一种关系型数据库（PostgreSQL）

## 决策

### 技术栈
**决策**: 使用Spring Boot 2.7.x + Java 8、Maven、PostgreSQL、Spring Security + JWT。

**理由**: 
- Spring Boot是Java后端开发的行业标准
- Java 8是企业中最广泛使用的LTS版本，公司当前使用此版本
- Spring Boot 2.7.x是支持Java 8的最新稳定版本
- Maven广泛使用且比Gradle更易于学习
- PostgreSQL是生产级数据库，在企业中很常见
- JWT身份认证对REST友好且广泛采用

**考虑的替代方案**: 
- Gradle（更强大但学习曲线更陡）
- MySQL（功能不如PostgreSQL丰富）
- 纯Spring MVC不用Boot（学习时样板代码太多）

### 项目结构
**决策**: 采用多阶段项目，逐步增加复杂度 - 阶段1：简单CRUD API，阶段2：添加身份认证，阶段3：添加业务逻辑和关联关系。

**理由**: 渐进式复杂度防止信息过载，允许在添加新概念前巩固现有知识。

**考虑的替代方案**: 单个复杂项目（一次性太多）、多个独立示例（缺乏连续性）。

### 学习路径架构
**决策**: 每个模块采用"理论+实践"模式 - 简要概念讲解后跟实践编码练习及解决方案。

**理由**: 前端开发者通过实践学习效果最好。理论提供背景，但实践建立肌肉记忆。

**考虑的替代方案**: 重理论方法（太抽象）、纯代码（缺少概念基础）。

### 数据库方法
**决策**: 使用JPA/Hibernate + Spring Data JPA仓储，避免原始JDBC或SQL。

**理由**: JPA是标准Java ORM，Spring Data消除样板代码，符合企业实践。

**考虑的替代方案**: JDBC（太底层）、MyBatis（不太常见，更偏SQL中心）。

### 测试策略
**决策**: JUnit 5 + Mockito用于单元测试，Spring Boot Test用于集成测试，专注于实用测试模式。

**理由**: 这些是行业标准工具。前端开发者理解测试概念，需要Java特定实现方式。

**考虑的替代方案**: TestNG（在Spring生态中不太常见）。

## 风险 / 权衡

**风险**: 同时接触太多新概念导致信息过载  
→ **缓解措施**: 清晰的阶段划分，每个模块的前置条件，逐步构建复杂度

**风险**: 学习练习缺乏实际场景  
→ **缓解措施**: 贯穿整个项目构建一个真实的应用（博客/任务管理系统），而不是零散的示例

**风险**: 开发环境设置障碍（JDK版本、IDE配置）  
→ **缓解措施**: 详细的环境设置指南，使用IntelliJ IDEA Community（最好的Java IDE，免费）

**风险**: 在Java特定概念上卡住（注解、依赖注入、OOP模式）  
→ **缓解措施**: 在适用情况下与JavaScript/前端概念对比，清晰的图解说明

**权衡**: 使用Spring Boot意味着在理解底层Java EE概念之前先学习Spring抽象  
→ **接受**: Spring Boot是公司实际使用的；如果需要可以后续学习基础知识

**权衡**: PostgreSQL需要安装 vs H2内存数据库  
→ **接受**: 生产就绪 > 便利性；H2用于快速测试，但PostgreSQL用于主要开发
