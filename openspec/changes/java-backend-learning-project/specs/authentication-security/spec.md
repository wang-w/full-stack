## 新增需求

### 需求: Spring Security配置
系统应教授添加Spring Security依赖、理解安全过滤器链以及配置身份认证/授权。

#### 场景: 启用Spring Security
- **当** 学习者添加spring-boot-starter-security依赖时
- **则** 应用程序应自动保护所有端点，要求身份认证

#### 场景: 自定义安全配置
- **当** 学习者自定义安全性时
- **则** 他们应创建带有SecurityFilterChain bean的@Configuration类，定义允许的端点和认证规则

### 需求: 使用JWT的用户身份认证
系统应实现基于JWT的身份认证，包括用户注册、登录、令牌生成和令牌验证。

#### 场景: 用户注册
- **当** 用户注册时
- **则** 系统应接受用户名/邮箱/密码，使用BCryptPasswordEncoder对密码进行哈希，并将用户存储在数据库中

#### 场景: 用户登录
- **当** 用户提供有效凭据时
- **则** 系统应验证用户身份，生成包含用户ID和角色的JWT令牌，并返回带有过期时间的令牌

#### 场景: 令牌验证
- **当** 请求在Authorization标头中包含JWT令牌时
- **则** 系统应验证令牌签名、检查过期时间、提取用户详细信息并设置SecurityContext

### 需求: JWT令牌结构
系统应创建包含标头、负载（声明：userId、roles、issued-at、expiration）和使用密钥的签名的JWT令牌。

#### 场景: 令牌生成
- **当** 系统生成JWT时
- **则** 令牌应包含主题（用户ID）、权限（角色）、签发时间戳、过期时间（例如24小时），并使用HS256签名

#### 场景: 令牌解析
- **当** 系统接收JWT令牌时
- **则** 它应解析令牌、使用密钥验证签名，并提取用于授权的声明

### 需求: 基于角色的授权
系统应实现使用@PreAuthorize、@Secured注解和方法级安全的基于角色的访问控制（RBAC）。

#### 场景: 端点授权
- **当** 学习者保护端点时
- **则** 他们应使用@PreAuthorize("hasRole('ADMIN')")或在SecurityFilterChain中使用antMatchers

#### 场景: 多角色访问
- **当** 端点允许多个角色时
- **则** 他们应使用@PreAuthorize("hasAnyRole('ADMIN', 'USER')")或SpEL表达式

### 需求: 密码安全
系统应使用BCrypt进行密码哈希，永远不存储明文密码，并实现安全的密码更新流程。

#### 场景: 密码哈希
- **当** 用户创建或更改密码时
- **则** 系统应使用BCryptPasswordEncoder对密码进行哈希，强度因子为默认值（10）

#### 场景: 密码验证
- **当** 用户登录时
- **则** 系统应使用PasswordEncoder.matches()将密码与存储的哈希进行验证

### 需求: 安全最佳实践
系统应实现CORS配置、无状态API的CSRF保护策略以及安全标头配置。

#### 场景: CORS配置
- **当** 前端发出跨域请求时
- **则** 系统应在CORS配置中配置允许的来源、方法和标头

#### 场景: 无状态安全
- **当** 使用JWT身份认证时
- **则** 系统应禁用CSRF保护，因为令牌提供了针对CSRF攻击的保护
