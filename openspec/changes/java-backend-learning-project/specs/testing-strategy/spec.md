## 新增需求

### 需求: 使用JUnit 5进行单元测试
系统应教授使用JUnit 5注解（@Test、@BeforeEach、@AfterEach）、断言和测试生命周期编写单元测试。

#### 场景: 编写基本单元测试
- **当** 学习者测试单个方法时
- **则** 他们应创建带有@Test注解的测试方法，使用assertEquals、assertTrue、assertNotNull断言

#### 场景: 测试设置和清理
- **当** 学习者需要测试初始化时
- **则** 他们应使用@BeforeEach在每个测试前进行设置，使用@AfterEach在每个测试后进行清理

### 需求: 使用Mockito进行Mock
系统应演示创建mock对象、存根方法调用、验证交互以及使用@Mock、@InjectMocks注解。

#### 场景: Mock依赖项
- **当** 学习者测试服务层时
- **则** 他们应使用@Mock模拟仓储，使用@InjectMocks注入到服务中，并使用when().thenReturn()进行存根

#### 场景: 验证方法调用
- **当** 学习者验证行为时
- **则** 他们应使用verify()来确保方法以预期参数和正确次数被调用

### 需求: Spring Boot集成测试
系统应教授使用@SpringBootTest、@WebMvcTest、MockMvc编写集成测试，通过Spring上下文测试控制器。

#### 场景: 控制器集成测试
- **当** 学习者测试REST端点时
- **则** 他们应使用@WebMvcTest(ControllerClass.class)，注入MockMvc，并执行带有期望的HTTP请求

#### 场景: 完整上下文集成测试
- **当** 学习者测试完整应用程序流程时
- **则** 他们应使用@SpringBootTest配合@AutoConfigureMockMvc来加载完整的应用程序上下文

### 需求: 仓储测试
系统应演示使用@DataJpaTest、测试数据库（H2）测试JPA仓储并验证查询。

#### 场景: 仓储方法测试
- **当** 学习者测试自定义仓储查询时
- **则** 他们应使用@DataJpaTest仅加载JPA组件，注入仓储，保存实体并断言查询结果

#### 场景: 测试实体关系
- **当** 学习者验证实体映射时
- **则** 他们应保存相关实体，获取它们，并验证级联操作和关系完整性

### 需求: 测试数据构建器
系统应教授创建测试数据构建器或fixture，用于可重用的测试实体创建。

#### 场景: 使用测试构建器
- **当** 学习者需要测试数据时
- **则** 他们应创建返回具有合理默认值的实体的构建器方法，允许特定测试的字段覆盖

#### 场景: 测试数据一致性
- **当** 多个测试需要相同的实体设置时
- **则** 他们应重用构建器方法以确保测试套件的一致性

### 需求: 测试覆盖率和最佳实践
系统应解释测试金字塔（多单元测试、一些集成测试、少量E2E）、测试边界以及测试什么与跳过什么。

#### 场景: 实现良好的覆盖率
- **当** 学习者编写测试时
- **则** 他们应专注于业务逻辑、边界情况、错误条件，避免测试框架代码或getter/setter

#### 场景: 测试命名和组织
- **当** 学习者组织测试时
- **则** 他们应使用描述性的测试名称（givenWhenThen或shouldDoSomethingWhen模式），对相关测试进行分组，每个测试保持一个断言焦点
