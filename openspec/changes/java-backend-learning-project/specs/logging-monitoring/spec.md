## 新增需求

### 需求: SLF4J和Logback配置
系统应使用SLF4J门面和Logback实现，配置日志级别并自定义日志格式。

#### 场景: 向类添加日志记录
- **当** 学习者需要日志记录时
- **则** 他们应使用@Slf4j（Lombok）或使用LoggerFactory.getLogger(ClassName.class)创建Logger实例

#### 场景: 配置日志级别
- **当** 学习者配置日志记录时
- **则** 他们应在application.properties中编辑logging.level.root、logging.level.com.example以设置包特定级别（TRACE、DEBUG、INFO、WARN、ERROR）

### 需求: 结构化日志记录实践
系统应教授日志记录最佳实践：记录什么（方法的进入/退出、状态更改、错误）、日志级别、参数化日志记录以及避免过度日志记录。

#### 场景: 选择日志级别
- **当** 学习者添加日志语句时
- **则** 他们应对需要关注的错误使用ERROR，对可恢复问题使用WARN，对重要事件使用INFO，对详细调试使用DEBUG，对细粒度使用TRACE

#### 场景: 参数化日志记录
- **当** 学习者使用变量记录日志时
- **则** 他们应使用log.info("User {} created with id {}", username, id)而不是字符串拼接以提高性能

### 需求: 请求日志记录
系统应使用过滤器或拦截器实现请求/响应日志记录，以使用关联ID跟踪API调用。

#### 场景: 记录HTTP请求
- **当** API请求到达时
- **则** 系统应记录HTTP方法、URI、请求参数和执行时间

#### 场景: 关联ID跟踪
- **当** 处理请求时
- **则** 系统应从标头生成或提取关联ID，在该请求的所有日志中包含它，并在响应标头中返回

### 需求: 应用程序监控基础
系统应介绍Spring Boot Actuator用于健康检查、指标和应用程序信息端点。

#### 场景: 启用Actuator
- **当** 学习者添加spring-boot-starter-actuator依赖时
- **则** 应用程序应公开/actuator/health、/actuator/info、/actuator/metrics端点

#### 场景: 健康检查配置
- **当** 学习者配置健康端点时
- **则** 他们应启用详细的健康信息并包含自定义健康指示器

### 需求: 性能监控
系统应教授使用Actuator指标、使用Micrometer的自定义指标以及识别瓶颈的基本性能跟踪。

#### 场景: 查看应用程序指标
- **当** 学习者检查指标时
- **则** 他们应访问/actuator/metrics以获取JVM指标、HTTP请求指标、数据库连接池统计信息

#### 场景: 创建自定义指标
- **当** 学习者需要业务指标时
- **则** 他们应注入MeterRegistry并创建Counter、Gauge或Timer以进行自定义测量

### 需求: 调试技术
系统应演示使用IDE调试器、设置断点、评估表达式以及使用Postman/curl调试REST API。

#### 场景: 调试应用程序
- **当** 学习者调查问题时
- **则** 他们应设置断点、以调试模式运行、单步执行代码、检查变量并评估表达式

#### 场景: 远程调试
- **当** 学习者调试正在运行的应用程序时
- **则** 他们应使用JVM参数启用远程调试，并将IDE调试器连接到正在运行的进程
