# Java后端开发环境配置指南

## 概述

本指南将帮助你从零开始配置Java后端开发环境。作为一名前端开发工程师，你将学习如何设置完整的Java开发工具链。

## 1. 安装JDK 8

### 什么是JDK？
JDK (Java Development Kit) 是Java开发工具包，包含编译器、运行时环境和开发工具。Java 8是目前企业中使用最广泛的长期支持版本(LTS)。

### 安装步骤

#### Windows:
1. 访问 [Adoptium](https://adoptium.net/) 或 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/)
2. 下载 JDK 8 for Windows (x64)
3. 运行安装程序，按默认选项安装
4. 配置环境变量：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 新建系统变量 `JAVA_HOME`，值为JDK安装路径（如：`C:\Program Files\Eclipse Adoptium\jdk-8.0.x`）
   - 编辑`Path`变量，添加 `%JAVA_HOME%\bin`

#### macOS:
```bash
# 使用Homebrew安装
brew install --cask temurin8

# 配置环境变量（添加到 ~/.zshrc 或 ~/.bash_profile）
export JAVA_HOME=$(/usr/libexec/java_home -v 1.8)
export PATH=$JAVA_HOME/bin:$PATH
```

#### Linux:
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-8-jdk

# CentOS/RHEL
sudo yum install java-1.8.0-openjdk-devel
```

### 验证安装
打开终端/命令提示符，运行：
```bash
java -version
javac -version
```

应该看到类似输出：
```
openjdk version "1.8.0_xxx"
javac 1.8.0_xxx
```

> **注意**: Java 8使用 `java -version`（单横杠），Java 9+使用 `java --version`（双横杠）。

## 2. 安装Maven

### 什么是Maven？
Maven是Java项目的构建工具和依赖管理工具，类似于前端的npm。它管理项目依赖、编译、测试和打包。

### 安装步骤

#### Windows:
1. 访问 [Maven官网](https://maven.apache.org/download.cgi)
2. 下载 `apache-maven-3.6.x-bin.zip`（或更高版本）
3. 解压到目录（如：`C:\Program Files\Apache\maven`）
4. 配置环境变量：
   - 新建系统变量 `MAVEN_HOME`，值为Maven解压路径
   - 编辑`Path`变量，添加 `%MAVEN_HOME%\bin`

#### macOS:
```bash
brew install maven
```

#### Linux:
```bash
# Ubuntu/Debian
sudo apt install maven

# CentOS/RHEL
sudo yum install maven
```

### 验证安装
```bash
mvn --version
```

应该看到Maven版本信息和Java版本。

### Maven镜像配置（可选，提升下载速度）

编辑 `~/.m2/settings.xml` (Windows: `C:\Users\你的用户名\.m2\settings.xml`):

```xml
<settings>
  <mirrors>
    <mirror>
      <id>aliyun</id>
      <mirrorOf>central</mirrorOf>
      <name>Aliyun Maven</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
```

## 3. 安装IntelliJ IDEA社区版

### 为什么选择IntelliJ IDEA？
IntelliJ IDEA是最强大的Java IDE，提供智能代码补全、重构、调试等功能。社区版免费且功能完整。

### 安装步骤

1. 访问 [JetBrains官网](https://www.jetbrains.com/idea/download/)
2. 下载 **Community Edition** (免费版)
3. 运行安装程序，选择以下选项：
   - ✅ 创建桌面快捷方式
   - ✅ 添加到PATH
   - ✅ 关联 .java 文件
   - ✅ 添加 "Open Folder as Project" 到右键菜单

### 首次启动配置

1. 启动IntelliJ IDEA
2. 选择主题（Darcula深色或Light浅色）
3. 跳过插件推荐（我们稍后手动安装）
4. 点击 "New Project" 测试

### 必装插件

打开 IntelliJ → Settings/Preferences → Plugins:

1. **Lombok** - 减少样板代码
2. **Spring Boot** (通常已内置)
3. **Maven Helper** - Maven依赖管理
4. **Rainbow Brackets** - 彩虹括号（可选，提升可读性）
5. **Chinese Language Pack** - 中文语言包（可选）

### 配置Java SDK

1. File → Project Structure → Project
2. Project SDK: 选择已安装的JDK 8
3. Project language level: 8

## 4. 安装PostgreSQL

### 什么是PostgreSQL？
PostgreSQL是一个功能强大的开源关系型数据库，企业级应用广泛使用。

### 方式A: 本地安装

#### Windows:
1. 访问 [PostgreSQL官网](https://www.postgresql.org/download/windows/)
2. 下载安装程序（推荐版本14或15）
3. 运行安装程序：
   - 记住设置的密码（用于postgres超级用户）
   - 端口保持默认 5432
   - 勾选pgAdmin 4（数据库管理工具）

#### macOS:
```bash
# 使用Homebrew
brew install postgresql@15
brew services start postgresql@15
```

#### Linux:
```bash
# Ubuntu/Debian
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 方式B: 使用Docker（推荐）

如果你已经熟悉Docker，这是最简单的方式：

```bash
docker run --name postgres-dev \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=java_learning_db \
  -p 5432:5432 \
  -d postgres:15
```

### 验证安装

使用pgAdmin或命令行连接：
```bash
psql -U postgres -h localhost
# 输入密码后应该能看到postgres提示符
```

## 5. 安装Docker Desktop（可选但推荐）

### 为什么需要Docker？
Docker可以让你轻松运行PostgreSQL、Redis等服务，无需复杂的本地安装。容器化也是现代应用部署的标准。

### 安装步骤

1. 访问 [Docker官网](https://www.docker.com/products/docker-desktop)
2. 下载适合你操作系统的Docker Desktop
3. 运行安装程序
4. 启动Docker Desktop
5. 完成初始化向导

### 验证安装
```bash
docker --version
docker-compose --version
```

### 基本Docker命令（前端开发者必知）

```bash
# 查看运行中的容器
docker ps

# 启动容器
docker start <容器名>

# 停止容器
docker stop <容器名>

# 查看容器日志
docker logs <容器名>
```

## 6. 安装Postman

### 什么是Postman？
Postman是API测试工具，你在前端开发中可能已经使用过。我们用它来测试后端REST API。

### 安装步骤

1. 访问 [Postman官网](https://www.postman.com/downloads/)
2. 下载并安装
3. 创建账号（可选，但推荐用于同步）

### 替代方案：使用curl

如果你习惯命令行：
```bash
# Windows (PowerShell已内置Invoke-WebRequest)
# 或安装curl for Windows

# macOS/Linux已预装curl
curl --version
```

## 7. 验证完整环境

创建一个测试脚本来验证所有工具：

### verify-environment.sh (macOS/Linux)
```bash
#!/bin/bash

echo "检查Java..."
java -version
echo ""

echo "检查Maven..."
mvn --version
echo ""

echo "检查Docker..."
docker --version
echo ""

echo "检查PostgreSQL..."
psql --version
echo ""

echo "✅ 环境检查完成！"
```

### verify-environment.ps1 (Windows PowerShell)
```powershell
Write-Host "检查Java..." -ForegroundColor Green
java -version
Write-Host ""

Write-Host "检查Maven..." -ForegroundColor Green
mvn --version
Write-Host ""

Write-Host "检查Docker..." -ForegroundColor Green
docker --version
Write-Host ""

Write-Host "✅ 环境检查完成！" -ForegroundColor Green
```

运行脚本，确保所有命令都能正常执行。

## 常见问题

### Q: JAVA_HOME配置后仍然找不到java命令？
A: 检查环境变量中的Path是否正确添加了`%JAVA_HOME%\bin`，重启终端或电脑。

### Q: Maven下载依赖很慢？
A: 配置Maven阿里云镜像（见上文Maven镜像配置）。

### Q: IntelliJ IDEA无法识别JDK？
A: File → Project Structure → Platform Settings → SDKs，手动添加JDK路径。

### Q: PostgreSQL安装失败？
A: 使用Docker方式运行PostgreSQL是最简单可靠的选择。

### Q: 我应该使用Windows、macOS还是Linux？
A: 都可以！但Linux/macOS对Java开发更友好。Windows用户建议使用WSL2 + Docker。
